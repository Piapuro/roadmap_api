### [C-1] JWT 検証が完全スルーになっている

**ファイル**: `middleware/supabase_auth.go:26-29`
**問題**: `Authorization: Bearer <token>` のヘッダー存在チェックのみで、**JWTの署名検証を行っていない**。任意のトークン文字列でも通過してしまう。

```go
// ❌ 現在のコード
token := strings.TrimPrefix(authHeader, "Bearer ")
_ = token
// TODO: validate JWT with Supabase secret and set user context
return next(c)
```

```go
// ✅ 修正例（#007 の本実装）
import "github.com/golang-jwt/jwt/v5"

claims := jwt.MapClaims{}
parsed, err := jwt.ParseWithClaims(token, claims, func(t *jwt.Token) (interface{}, error) {
    if _, ok := t.Method.(*jwt.SigningMethodHMAC); !ok {
        return nil, fmt.Errorf("unexpected signing method: %v", t.Header["alg"])
    }
    return []byte(m.jwtSecret), nil
})
if err != nil || !parsed.Valid {
    return c.JSON(http.StatusUnauthorized, map[string]string{"error": "invalid token"})
}
// userID を context に詰めて後続 middleware/handler で使えるようにする
c.Set("user_id", claims["sub"])
return next(c)
```

**理由**: Sprint 2〜4 で実装するすべての認証必須エンドポイントが、現状では未認証アクセスを受け付ける。#007 は2時間見積もりで Sprint 1 に入っているので **最優先で実装が必要**。

## 🟡 Minor Issues

### [m-1] abac_rules の seed データに論理的な誤りの可能性

**ファイル**: `sql/schema.sql:311`

```sql
(2, '自タスクのみ更新', 'task:update', '{"assigned_user_id": "self"}', 'deny', 20),
```

`effect: 'deny'` かつ条件 `assigned_user_id: "self"` だと「**自分に割り当てられたタスクの更新を拒否**」という意味になる。意図は逆のはず（他人のタスクを更新できないようにする）。

ABAC の実装時に条件の評価ロジックと合わせて再確認が必要。
